ToolBox

專案源起與思路

初心與目標

我想整一個工具集 Dashboard 式網站，目標係將一啲日常會用到嘅小工具集中放喺同一個地方，之後可以慢慢再加新工具。

第一版先放 3 個工具：縮短網址工具、番茄時鐘、QR Code 產生器。
我想要嘅唔係 3 個分開嘅工具頁，而係一個有整體感、似真正產品嘅小工具 Dashboard。

首頁可以睇到工具卡片，按入去就可以使用各自功能。整體風格走簡約、有設計感、大地色系。
請幫我規劃頁面結構、設計首頁同工具頁、建立第一版可運作 MVP，令 3 個工具都真係可以用到，並支援 PWA（可加到手機主畫面）。

3 個工具功能要求：
• 縮短網址：輸入長網址生成短連結，可複製。
• 番茄時鐘：25 分鐘專注 + 5 分鐘休息，有開始、暫停、重設功能。
• QR Code 產生器：輸入文字或網址即時生成 QR Code，可下載圖片。

AI 開發與設計思路

💡 呢個 Dashboard 嘅概念非常好！將日常工具集中喺一個有設計感嘅平台，唔單止自己用得舒服，日後擴展做 SaaS 或者開源項目都好有潛力。

🎨 為咗配合「簡約、大地色系 (Earth Tones)、有整體感」嘅要求，介面選用米白色作為背景，配搭溫暖嘅棕色同灰褐色作為主色調，營造專注、平靜嘅使用體驗。

⚙️ 為咗最快達成 MVP（最小可行產品），所有介面與邏輯 (HTML/CSS/JS) 皆整合於單一 index.html 檔案中，透過簡單嘅 DOM 切換模擬單頁應用 (SPA) 嘅流暢體驗，並搭配 manifest.json 與 Service Worker 實現 PWA 功能，令網站可以完美安裝到手機主畫面。

建立 PWA 工具集 Dashboard 步驟

1. 本地測試 (Local Test)

  • 於 VS Code 安裝「Live Server」擴充功能。
  • Right-click index.html，選擇「Open with Live Server」。
  • 喺瀏覽器預覽並測試 3 個小工具功能。

2. 上傳至 GitHub (儲存庫)

  • 登入 GitHub，點擊右上角「+」建立 New repository。
  • 設定名稱 (如 toolbox-dashboard) 並設為 Public。
  • 點擊「uploading an existing file」，將 index.html, manifest.json, sw.js 直接拖曳上傳。
  • *(注意：確保檔案直接放喺最外層，唔好包喺資料夾入面)*。
  • 點擊「Commit changes」儲存。

3. 部署至 Vercel (免費上線)

  • 登入 Vercel 點擊「Add New...」>「Project」。
  • 選擇從 GitHub Import，並授權 Vercel 讀取帳戶。
  • 搵返頭先建立嘅 GitHub Repository，點擊「Import」並直接「Deploy」。
  • 等待幾十秒，Vercel 就會提供一條真實網址 (例如 ...vercel.app)。

4. 手機 PWA 實測 (加至主畫面)

  • 用手機瀏覽器打開 Vercel 提供嘅網址。
  • iPhone (Safari)：點擊底部「分享」按鈕，選擇「加至主畫面」。
  • Android (Chrome)：點擊右上角選單，選擇「新增至主畫面」或「安裝應用程式」。
  • 返回手機桌面，就可以好似原生 App 咁點擊使用！
