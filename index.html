📝 Prompt 基礎概念

AI 應用實戰筆記

💬 什麼是 Prompt？

Prompt（中文：提示詞），就係你喺 AI 輸入框打入嘅所有文字。

無論係問題、請求、指示、對話——通通都叫 Prompt！
🔍 AI關鍵字： PROMPT

✌️ Prompt 分兩種

大部分時候，你可以好隨意咁問，就好似同朋友 WhatsApp 傾偈咁，AI 一樣識答你！所以我會將 Prompt 分成兩種用法：

1️⃣隨意 Prompt → 即興問問題

例如：

  • 「雞蛋點煮好食啲？」
  • 「日本最長嘅河叫咩名？」
  • 「點煮好味啲公仔麵？」
  • 「世界上最快嘅動物係咩？」
  • 「Python 係咩嚟？」

呢啲問題 AI 都可以秒速答你，因為只係搵資料嚟答你。
（細細聲講，其實我平時都係咁用，90% 都係即興問 😆）

✅ 用途：搵知識、快問快答、解決即時疑問。

2️⃣精準 Prompt → 比 AI 做深入任務

如果你想 AI 幫你做啲深入啲、複雜啲嘅任務，例如：

  • 寫文章
  • 給創作靈感
  • 比專業建議
  • 分析數據／報告
  • 設計內容框架…
🎯 就要學識寫精準 Prompt！
分類 說明 例子 適用場景
隨意 Prompt 即興輸入、唔需特別結構，AI 都能理解回應
  • 雞蛋點煮好食啲？
  • Python 係咩？
  • 世界最快動物係咩？
 快問快答、資料查詢、日常應用
精準 Prompt 有清晰指令、目標、語氣要求，適合進行深入任務 「請幫我寫一篇約 500 字嘅文章，主題係『環保生活建議』。語氣要親切易明，好似同朋友傾偈咁，分段清楚，開頭要有吸引句，結尾要有實用小總結。」 寫作、分析、創作、專業協作任務

精準 Prompt 核心

將你諗緊嘅嘢 ➜ 變成 AI 明嘅說話

即係話，你要將你腦入面嘅想法、目標、要求，轉化成清晰、具體、指令式嘅語句，先令 AI 幫得上手！

✅ 精準 Prompt 嘅兩大原則

1
足夠資訊

你要 AI 扮演邊個？針對邊個對象？有冇提供資料背景？

2
清晰指引

你想要邊類型回應？語氣係點？長短幾多？格式有冇要求？

📌 所以要記住 8 字真言：

「足夠資訊，清晰指引」

呢個，就係所有 Prompt 技巧嘅核心原則！

教學指南 PWA

🛠️ PWA 工具集 Dashboard 建立教學

💡

初心與目標： 呢個 Dashboard 嘅概念非常好！將日常工具集中喺一個有設計感嘅平台，唔單止自己用得舒服，日後擴展做 SaaS 或者開源項目都好有潛力。

設計思路

為咗配合你要求嘅「簡約、大地色系 (Earth Tones)、有整體感」，我哋會用米白色作為背景，配搭溫暖嘅棕色同灰褐色作為主色調。為咗令你可以最快測試呢個 MVP（最小可行產品），我會將 HTML、CSS 同 JavaScript 寫喺同一個 index.html 檔案入面，並透過簡單嘅 DOM 切換嚟模擬單頁應用（SPA）嘅流暢體驗。另外會提供 PWA 必需嘅 manifest.json 同 Service Worker。

📁 專案檔案結構

請喺你嘅電腦開一個新資料夾（例如 tool-dashboard），然後建立以下 3 個檔案：

  • index.html (主程式、UI 同邏輯)
  • manifest.json (PWA 設定)
  • sw.js (Service Worker，令 PWA 支援離線快取)

1. 核心代碼：index.html

呢個檔案包含咗大地色系嘅 CSS 設計、Dashboard 卡片佈局，以及 3 個工具嘅完整運作邏輯。（代碼已在主程式運作中）

2. PWA 設定檔：manifest.json

呢個檔案會話畀瀏覽器知，呢個網站可以安裝成為手機 App。
(備註：icon 我暫時借用咗一張網上免費圖示作示範，日後你可以換成自己設計嘅 512x512 PNG 圖)

{ "name": "ToolBox Dashboard", "short_name": "ToolBox", "description": "日常小工具 Dashboard，包含縮短網址、番茄時鐘同 QR Code 產生器。", "start_url": "/index.html", "display": "standalone", "background_color": "#f4f1ea", "theme_color": "#8b7355", "icons": [ { "src": "https://cdn-icons-png.flaticon.com/512/2885/2885412.png", "sizes": "512x512", "type": "image/png" } ] }

3. PWA 離線快取：sw.js

加入 Service Worker 可以令你嘅網站喺手機加落主畫面後，即使網絡唔穩定都可以秒速開啟 UI。

const CACHE_NAME = 'toolbox-cache-v1'; const urlsToCache = [ './index.html', './manifest.json' ]; self.addEventListener('install', event => { event.waitUntil( caches.open(CACHE_NAME) .then(cache => { return cache.addAll(urlsToCache); }) ); }); self.addEventListener('fetch', event => { event.respondWith( caches.match(event.request) .then(response => { // 如果有 cache 就用 cache，冇就正常 fetch return response || fetch(event.request); }) ); });

🚀 點樣運行同測試 PWA？

因為瀏覽器嘅安全限制（CORS 同 Service Worker 規定），你唔可以直接雙擊 index.html 開啟來測試 PWA 功能。你需要一個 Local Server：

1️⃣

喺自己電腦做本地測試 (Local Test)

  • 最簡單做法 (使用 VS Code)： 如果你用 Visual Studio Code 編輯器，請去左側 Extension (擴充功能) 搜尋並安裝 Live Server。
  • 安裝後，Right-click 你個 index.html，選擇 "Open with Live Server"。
  • 瀏覽器會自動彈出一個類似 http://127.0.0.1:5500 嘅網址，你就可以喺電腦完美測試 3 個小工具嘅功能。開啟後，用 Chrome 瀏覽器睇，網址列右側應該會出現一個「安裝 (Install)」嘅圖示，撳落去就可以好似真正 App 咁安裝落電腦或模擬器手機嘅主畫面。
2️⃣

免費部署上網 (Deploy to Web)

要將個網站變成真正可以分享畀人、又可以裝落手機主畫面嘅 PWA，你需要將檔案放上一個支援 HTTPS 加密嘅真實網絡空間。強烈推薦以下免費平台：

第一步：建立 GitHub Repository (儲存庫)

  • 如果你未有 GitHub 戶口，請先去 github.com 註冊一個免費戶口。
  • 登入 GitHub 後，喺右上角頭像隔籬撳 + 號，揀 "New repository"。
  • Repository name: 隨便改個名，例如 toolbox-dashboard。
  • 揀 "Public"，然後撳最下面嘅 "Create repository"。

第二步：上傳你嗰 3 個檔案

  • 見到 "or create a new repository on the command line" 嗰段字，上面有行細字寫住 "uploading an existing file"，撳嗰條藍色 link。
  • 將你嗰 3 個檔案 (index.html, manifest.json, sw.js) 直接拖入去個框入面。
  • 拉到底下 "Commit changes"，寫返一句描述（例如 Initial commit），然後撳綠色按鈕 "Commit changes"。

第三步：喺 Vercel 連接 GitHub

  • 去 vercel.com 註冊免費戶口。喺 Vercel 嘅 Dashboard，搵到 "Import Project" 或 "Add New -> Project"。
  • 佢會問你 "From where?"，選擇 "GitHub" 並授權讀取你嘅帳戶。
  • 喺清單入面搵到 toolbox-dashboard，撳 "Import"。其他設定直接撳 "Deploy" 就得，Vercel 會自動偵測到你係 HTML 項目。

之後點樣更新網站？
你喺電腦改完 index.html 或者加新功能，重新 Upload 上 GitHub。Vercel 會喺幾秒鐘後自動感應到，自動 Deploy 成個網。

3️⃣

手機 PWA 實測 (Mobile Test)

當你攞到真實網址後，就可以用手機感受真正嘅「產品體驗」：

  • 用手機瀏覽器（iPhone 建議用 Safari，Android 用 Chrome）打開你嘅網址。
  • iPhone 用戶： 撳畫面底部的「分享」掣（正方形有向上箭嘴），然後慢慢向下拉，揀 「加至主畫面 (Add to Home Screen)」。
  • Android 用戶： 瀏覽器通常會主動彈出「新增至主畫面」或「安裝應用程式」嘅提示；如果冇，可以撳右上角三點選單揀安裝。
  • 裝完之後返回手機桌面，打開佢，你會發現上面冇咗瀏覽器嘅網址列，就好似一個原生 App 咁！
⚠️ 一個好重要嘅檢查：
當你部署完，Vercel 會畀一條網址你。請一定要用手機瀏覽器打開呢條連結試下，睇下個介面夠唔夠簡約，試下撳「加入主畫面」。如果發現功能用唔到，請喺瀏覽器撳 F12 (開發者工具) -> Console，睇下有冇紅色嘅錯誤訊息。
